erikfeng.dev — Setup Guide #

Your blog/essay system: Hugo + Typo theme + GitHub Pages + Buttondown + Cloudflare domain.

Total cost: ~$13/year (domain only)

Overview #

Component Purpose Cost
erikfeng.dev Domain ~$13/year
Hugo + Typo Static site generator + theme Free
GitHub Pages Hosting Free
Buttondown Email subscriptions (public + private essays) Free (up to 100 subscribers)
Cloudflare DNS + CDN Free

How it works:

  1. You write an essay in markdown
  2. Push to GitHub → site auto-deploys
  3. If public: appears on erikfeng.dev AND gets emailed to all subscribers
  4. If private: emailed only to “premium” (inner circle) subscribers via Buttondown, never touches the website

Step 1: Register the Domain #

  1. Go to domains.cloudflare.com
  2. Create a Cloudflare account if you don’t have one
  3. Search for erikfeng.dev
  4. Register it (~$13/year)

Cloudflare charges wholesale — no markup, no inflated renewals.

Also: Set a backorder on park.io for erikfeng.com (~$10-20 if they grab it). Check back periodically — current registration expires May 2026.

Step 2: Set Up Hugo Locally #

Install Hugo #

macOS:

brew install hugo

Windows:

choco install hugo-extended
# or
winget install Hugo.Hugo.Extended

Linux:

sudo apt install hugo
# or snap install hugo

Verify: hugo version — you need v0.116.0 or later for Typo.

Create Your Site # 

hugo new site erikfeng.dev --config toml
cd erikfeng.dev

Install the Typo Theme # 

git init
git submodule add --depth=1 https://github.com/tomfran/typo.git themes/typo

Configure the Site #

Replace the contents of hugo.toml with:

baseURL = "https://erikfeng.dev/"
languageCode = "en"
title = "Erik Feng"
theme = "typo"

[params]
# Collection to display on home page
homeCollectionTitle = "Essays"
homeCollection = "essays"

# Lists
paginationSize = 20
listSummaries = true
listDateFormat = "2 Jan 2006"

# Breadcrumbs
[params.breadcrumbs]
enabled = true
showCurrentPage = true
home = "Home"

# Social icons
[[params.social]]
name = "github"
url = "https://github.com/YOUR_GITHUB_USERNAME"

[[params.social]]
name = "instagram"
url = "https://instagram.com/erikkfeng"

# If/when you set up Buttondown:
# [[params.social]]
# name = "email"
# url = "https://buttondown.com/erikfeng"

# Menu
[[params.menu]]
name = "home"
url = "/"

[[params.menu]]
name = "essays"
url = "/essays"

[[params.menu]]
name = "about"
url = "/about"

# Syntax highlighting (if you ever write code posts)
[markup]
[markup.highlight]
style = "algol"

mkdir -p content

Test Locally # 

hugo server -D

Open http://localhost:1313 — you should see your site with the Typo theme.

Step 3: Deploy to GitHub Pages #

Create a GitHub Repository #

  1. Go to github.com/new
  2. Name it erikfeng.dev (or whatever you prefer)
  3. Make it public (required for free GitHub Pages)
  4. Don’t initialize with README

Push Your Site # 

# From your site directory
git remote add origin git@github.com:YOUR_USERNAME/erikfeng.dev.git
git add .
git commit -m "Initial site"
git push -u origin main

Add GitHub Actions Workflow #

Create .github/workflows/hugo.yml:

name: Deploy Hugo site to Pages

on:
  push:
    branches: ["main"]

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: false

defaults:
  run:
    shell: bash

jobs:
  build:
    runs-on: ubuntu-latest
    env:
      HUGO_VERSION: 0.141.0
    steps:
      - name: Install Hugo CLI
        run: |
          wget -O ${{ runner.temp }}/hugo.deb https://github.com/gohugoio/hugo/releases/download/v${HUGO_VERSION}/hugo_extended_${HUGO_VERSION}_linux-amd64.deb \
          && sudo dpkg -i ${{ runner.temp }}/hugo.deb

      - name: Checkout
        uses: actions/checkout@v4
        with:
          submodules: recursive
          fetch-depth: 0

      - name: Setup Pages
        id: pages
        uses: actions/configure-pages@v5

      - name: Build with Hugo
        env:
          HUGO_CACHEDIR: ${{ runner.temp }}/hugo_cache
          HUGO_ENVIRONMENT: production
          TZ: Europe/London
        run: |
          hugo \
            --gc \
            --minify \
            --baseURL "${{ steps.pages.outputs.base_url }}/"

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./public

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4

Enable GitHub Pages #

  1. Go to your repo → Settings → Pages
  2. Under “Build and deployment”, select GitHub Actions as the source

Push the Workflow # 

git add .
git commit -m "Add GitHub Actions deploy"
git push

Your site should now be live at https://YOUR_USERNAME.github.io/erikfeng.dev/ within a few minutes.

Step 4: Connect Your Domain #

In Cloudflare (DNS): #

Add these records:

Type Name Content
CNAME erikfeng.dev YOUR_USERNAME.github.io
CNAME www YOUR_USERNAME.github.io

In GitHub: #

  1. Go to repo → Settings → Pages → Custom domain
  2. Enter erikfeng.dev
  3. Check “Enforce HTTPS”

GitHub will automatically provision an SSL certificate. This can take up to 24 hours but usually happens in minutes.

Update hugo.toml: #

Make sure baseURL is set to:

baseURL = "https://erikfeng.dev/"

Commit and push. Your site is now live at https://erikfeng.dev.

Step 5: Set Up Buttondown #

  1. Go to buttondown.com and create an account
  2. The free tier supports up to 100 subscribers — more than enough

Configure Tiers: #

  • Free subscribers = public (anyone can sign up)
  • Premium subscribers = your inner circle (you manually add them or send invite links)

Note: “Premium” doesn’t mean paid — you just won’t enable payments.

Add a Subscribe Link to Your Site: #

Update your hugo.toml to uncomment the email social link:

[[params.social]]
name = "email"
url = "https://buttondown.com/erikfeng"

You can also add a subscribe note to your About page.

Publishing Workflow: #

Public essay:

  1. Write in markdown, save to content/essays/
  2. git add . && git commit -m "New essay: [title]" && git push
  3. Site deploys automatically
  4. Copy the essay into Buttondown and send to all subscribers

Private essay:

  1. Write in markdown (don’t push to GitHub)
  2. Paste into Buttondown
  3. Send to premium subscribers only
  4. Optionally keep the .md file locally for your own records

Your Writing Workflow (Day-to-Day) # 

1. Have an idea
2. Create content/essays/some-slug.md
3. Write in markdown
4. hugo server -D       ← preview locally
5. Set draft: false when ready
6. git add . && git commit -m "essay: title" && git push
7. (Optional) Send via Buttondown for email subscribers

That’s it. Write, push, done.

Future Expansion #

When you’re ready to turn this into a research portfolio:

  1. Add new sections to content/ (e.g., content/research/, content/publications/)
  2. Add menu items to hugo.toml
  3. The essays stay exactly where they are

No migration, no new platform, no new domain. Just more content.
